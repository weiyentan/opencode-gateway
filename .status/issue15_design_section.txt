## Design decisions (from grill session)

### Tooling
- **Python:** 3.12
- **Package manager:** pip + `requirements.txt` (relaxed ranges like `fastapi>=0.115.0,<1.0.0`)
- **Build config:** `pyproject.toml` (for ruff, pytest, mypy tooling)

### Application architecture
- **App creation:** `create_app()` factory function in `app/core/factory.py`
- **Lifespan:** `@asynccontextmanager` pattern (startup/shutdown hooks)
- **Settings:** Module global in `app/core/config.py` + FastAPI dependency injection for route overrides
- **Entry points:** Both `app/__main__.py` (`python -m app` for dev) and `uvicorn app.main:app` (for production)

### Project structure (initial)
```
app/
├── __init__.py
├── __main__.py          # dev entry point (uvicorn.run)
├── main.py              # prod entry point (app instance)
├── api/
│   └── __init__.py      # route stubs
├── core/
│   ├── __init__.py
│   ├── factory.py       # create_app()
│   └── config.py        # Settings (Pydantic BaseSettings)
├── db/
│   ├── __init__.py
│   └── session.py       # stub — will hold connection pool later
├── executors/           # stub — created but empty
│   └── __init__.py
└── opencode/            # stub — created but empty
    └── __init__.py
```

### Acceptance criteria (refined)
- [ ] Project directory structure matches the layout above
- [ ] `pyproject.toml` with Python 3.12 target and tooling config
- [ ] `requirements.txt` with FastAPI, uvicorn, pydantic, asyncpg (relaxed ranges)
- [ ] `Settings` class in `core/config.py` using Pydantic BaseSettings
- [ ] `create_app()` factory in `core/factory.py` with `@asynccontextmanager` lifespan
- [ ] `app/__main__.py` calls `uvicorn.run()` with settings
- [ ] `app/main.py` exports the app instance via the factory
- [ ] App starts without errors on `python -m app` and `uvicorn`
- [ ] Port is configurable via environment variable (default 8000)
